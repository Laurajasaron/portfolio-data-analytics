01
Problème

Un e-commerce qui croît sans comprendre ses clients

ShopNova, un e-commerce de décoration intérieure basé en France, constate une stagnation de son chiffre d'affaires malgré une base clients en augmentation constante. Les équipes marketing envoient les mêmes campagnes à tous les clients, sans distinction. Résultat : un taux de désabonnement élevé et un budget publicitaire sous-optimisé.

Question centrale : comment identifier les clients à fort potentiel pour concentrer les efforts marketing là où ils génèrent le plus de valeur ?

L'hypothèse de travail est que la base clients est hétérogène — certains clients génèrent l'essentiel du CA, tandis qu'une large proportion est inactive depuis plusieurs mois. Une segmentation comportementale permettrait d'adresser chaque groupe avec la bonne action au bon moment.

02
Données

Nettoyage et construction de la table RFM

Source : Online Retail Dataset — UCI Machine Learning Repository. 541 909 transactions de novembre 2010 à décembre 2011, 8 variables (CustomerID, InvoiceDate, Quantity, UnitPrice, Country…). Première étape : suppression des valeurs nulles et des retours produits (quantités négatives).

rfm_analysis.py Python 
import pandas as pd
import numpy as np

# Chargement et nettoyage initial
df = pd.read_excel('online_retail.xlsx')
df.dropna(subset=['CustomerID'], inplace=True)
df = df[df['Quantity'] > 0]
df['TotalPrice'] = df['Quantity'] * df['UnitPrice']

# Construction de la table RFM
snapshot = pd.Timestamp('2011-12-11')
rfm = df.groupby('CustomerID').agg({
    'InvoiceDate': lambda x: (snapshot - x.max()).days,
    'InvoiceNo'  : 'nunique',
    'TotalPrice' : 'sum'
})
rfm.columns = ['Recency', 'Frequency', 'Monetary']
cleaning.sql SQL 
-- Extraction des transactions valides
SELECT
    CustomerID,
    COUNT(DISTINCT InvoiceNo)  AS Frequency,
    SUM(Quantity * UnitPrice)  AS Monetary,
    MAX(InvoiceDate)           AS LastPurchase
FROM transactions
WHERE Quantity > 0
  AND CustomerID IS NOT NULL
GROUP BY CustomerID
ORDER BY Monetary DESC;

Après nettoyage : 4 372 clients retenus sur les 4 373 uniques (suppression d'un outlier B2B avec >200K€ de CA atypique).

03
Solution

Clustering K-Means : 4 segments distincts

Après normalisation des 3 dimensions RFM (StandardScaler), l'algorithme K-Means identifie le nombre optimal de clusters via la méthode du coude (inertie). 4 clusters offrent le meilleur compromis variance inter/intra.

clustering.py Python · scikit-learn 
from sklearn.preprocessing import StandardScaler
from sklearn.cluster import KMeans
import matplotlib.pyplot as plt

# Normalisation
scaler    = StandardScaler()
rfm_scaled = scaler.fit_transform(rfm)

# Méthode du coude — choix de k=4
inertias = []
for k in range(2, 10):
    km = KMeans(n_clusters=k, random_state=42)
    km.fit(rfm_scaled)
    inertias.append(km.inertia_)

# Application du modèle final
kmeans = KMeans(n_clusters=4, random_state=42)
rfm['Segment'] = kmeans.fit_predict(rfm_scaled)
rfm.to_csv('rfm_segments.csv')

04
Recommandations

4 actions prioritaires actionnables

R1

Créer un programme de fidélité à 3 niveaux (Bronze / Silver / Gold) ciblé sur les Champions et les Fidèles à risque. Objectif : augmenter la fréquence d'achat moyenne de 15% en 6 mois grâce à des avantages exclusifs (accès anticipé aux nouvelles collections, livraison offerte).

R2

Automatiser une séquence email de réactivation en 3 temps pour les Fidèles à risque : J+30 (contenu inspirationnel), J+60 (coupon 10%), J+90 (coupon 20% + dernier contact avant passage en Inactif). ROI estimé : réactiver 22% de ce segment.

R3

Concentrer le budget retargeting sur les Occasionnels via des publicités personnalisées par catégorie produit déjà achetée. Offrir un incentive sur le 2e achat (livraison offerte ou -15%) pour ancrer l'habitude d'achat.

R4

Nettoyer la base Inactifs (18% des contacts, 5% du CA) : supprimer les profils sans engagement depuis 12 mois. Économie directe estimée à -18% sur les coûts d'emailing, libérant du budget pour les segments à fort potentiel.
